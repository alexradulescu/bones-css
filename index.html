What? Why? Another framework?

Bones is a super simple and small css framework.
It was inspired by Skeleton css, Water.css and other super small and focused mini or micro-frameworks(if we can even call them such).
Bones works directly with html elements, no css classes, no ids, nothing. This has some pros and cons, or rather some use-cases and some stay-away-cases ;-) .
It is almost what could have been if browsers cared to make good-looking standards.
I made it for one specific case: building a simple, quick, beautiful prototype based solely on good HTML code.
I make a lot of prototypes to play with different things, and I got bored to keep adding lots of classes and bigfiles. This just works with one line import.
At the same time, this is absolutely not a for a full on serious app or website. It will clash with most other frameworks and would only work if you want to manualy extend it.
Also it is super oppinionated. Some choises are obvious, while others needed some serious code. Feel free to check the comments.
Last but not least, hope it helps, from me a message if you used it with a screenshot or with suggestions and check the examples below.
